# 🛋️ SofaStream

**SofaStream** — это веб-платформа для совместного просмотра видеоконтента в реальном времени, созданная с фокусом на пары, находящиеся на расстоянии. 

В отличие от трансляции экрана в мессенджерах, SofaStream обеспечивает **жесткую синхронизацию состояния плеера** у всех участников комнаты, компенсируя сетевые задержки и автоматически реагируя на буферизацию (падение скорости интернета) у отдельных пользователей.

*(🚧 Проект находится в стадии активной разработки 🚧)*

## Архитектура

Проект спроектирован с упором на качество кода, тестируемость и масштабируемость. В основе лежит **Clean Architecture** и принципы **Domain-Driven Design (DDD)**.

### Синхронизация времени
Для решения проблемы рассинхрона состояние плеера хранится не как статичная позиция, а как вектор: `[PlaybackState, CurrentPosition, LastUpdatedAt]`. 
Если у одного из участников (например, из-за слабого Wi-Fi) начинается буферизация, комната переходит в состояние `Buffering` для всех, и воспроизведение автоматически продолжается только тогда, когда отстающий участник загрузит чанк видео.

### Архитектурные паттерны
- **Rich Domain Model:** Бизнес-правила и инварианты комнаты (смена хоста, идемпотентность участников, логика буферизации) надежно инкапсулированы внутри Aggregate Root (`Room.cs`).
- **CQRS:** Потоки чтения и изменения состояния строго разделены с помощью `MediatR`. Это избавляет систему от раздутых сервисов (God Objects).
- **Domain Events:** Изменение состояния домена генерирует события (например, `RoomPlaybackStateChangedEvent`), которые публикуются только после успешной фиксации транзакции в БД. Это обеспечивает слабую связность (Loose Coupling) между ядром и инфраструктурой веб-сокетов.

## 🛠 Стек технологий
- **Backend:** C# 14, .NET 10, ASP.NET Core (Web API)
- **Architecture:** Clean Architecture, DDD, CQRS (MediatR)
- **Database:** PostgreSQL, Entity Framework Core 
- **Real-time Communication:** SignalR (WebSockets)
- **Frontend:** *В планах (React / Vue.js)*
